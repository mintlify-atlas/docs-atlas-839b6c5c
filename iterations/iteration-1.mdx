## Test Cases to Implement

Open `src/pages/home/__tests__/Home.test.tsx` and implement this test:

```typescript
describe('Iteración 1 - Listado de categorías', () => {
  it.todo('should render the list of categories in the navigation')

```

<Steps>

### Change the test from todo to active

Replace `it.todo` with `it` and add the test implementation:

```typescript
it('should render the list of categories in the navigation', async () => {
  render(<App />)
  
  // Wait for categories to load
  const categoryItems = await screen.findAllByRole('listitem')
  
  // Verify all categories are rendered
  expect(categoryItems).toHaveLength(3)
  expect(screen.getByText('Fruta y verdura')).toBeInTheDocument()
  expect(screen.getByText('Cereales')).toBeInTheDocument()
  expect(screen.getByText('Aperitivos')).toBeInTheDocument()
})
```

### Run the test and see it fail (RED)

The test will fail because the categories aren't being fetched or rendered yet.

```bash
npm test
```

### Implement the minimum code to make it pass (GREEN)

In your navigation component, fetch and display the categories:

```typescript
import { useEffect, useState } from 'react'
import type { Category } from '../types'

const Navigation = () => {
  const [categories, setCategories] = useState<Category[]>([])

  useEffect(() => {
    fetch('/categories')
      .then(res => res.json())
      .then(data => setCategories(data))
  }, [])

  return (
    <nav className="navigation">
      <ul className="navigation__list">
        {categories.map(category => (
          <li key={category.id} className="navigation__list-item">
            {category.displayName}
          </li>
        ))}
      </ul>
    </nav>
  )
}
```

### Run the test again - it should pass (GREEN)

```bash
npm test
```

### Refactor if needed

Consider extracting the fetch logic to a custom hook or moving it to a separate function. The important thing is that tests continue passing after any refactor.

</Steps>

## Implementation Hints

<Accordion title="How do I fetch data in React?">

Use the `useEffect` hook to fetch data when the component mounts:

```typescript
useEffect(() => {
  fetch('/categories')
    .then(res => res.json())
    .then(data => setCategories(data))
}, [])
```

The empty dependency array `[]` ensures the fetch only happens once on mount.

</Accordion>

<Accordion title="How do I render a list in React?">

Use the `map` function to transform an array of data into React elements:

```typescript
{categories.map(category => (
  <li key={category.id} className="navigation__list-item">
    {category.displayName}
  </li>
))}
```

Always provide a unique `key` prop when rendering lists.

</Accordion>

<Accordion title="Why use async/await in tests?">

MSW handlers are asynchronous. Use `findAllByRole` or `findByText` (instead of `getBy*`) to wait for elements to appear after the API call completes:

```typescript
const categoryItems = await screen.findAllByRole('listitem')
```

</Accordion>

## Common Issues

<Tip>
**Issue:** Test times out or categories don't appear

**Solution:** Make sure you're using `findAllByRole` (async) instead of `getAllByRole` (sync). The categories need time to load from the mocked API.
</Tip>

<Tip>
**Issue:** "Cannot read property 'map' of undefined"

**Solution:** Initialize your state with an empty array: `useState<Category[]>([])`
</Tip>
