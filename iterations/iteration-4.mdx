### Create the ViewModeContext

Create a new file `src/contexts/ViewModeContext.tsx`:

```typescript
import { createContext, useContext, useState } from 'react'

type ViewMode = 'card' | 'list'

interface ViewModeContextType {
  viewMode: ViewMode
  toggleViewMode: () => void
}

const ViewModeContext = createContext<ViewModeContextType | null>(null)

export const ViewModeProvider = ({ children }: { children: React.ReactNode }) => {
  const [viewMode, setViewMode] = useState<ViewMode>('card')
  
  const toggleViewMode = () => {
    setViewMode(prev => prev === 'card' ? 'list' : 'card')
  }

  return (
    <ViewModeContext.Provider value={{ viewMode, toggleViewMode }}>
      {children}
    </ViewModeContext.Provider>
  )
}

export const useViewMode = () => {
  const context = useContext(ViewModeContext)
  if (!context) {
    throw new Error('useViewMode must be used within a ViewModeProvider')
  }
  return context
}
```

## Understanding Context API

<Note>
The Context API solves the "prop drilling" problem by allowing you to share state across components without passing props through every level.

**Three key parts:**
1. **Context creation:** `createContext()`
2. **Provider:** Wraps your component tree and provides the value
3. **Consumer:** Uses `useContext()` to access the value
</Note>

## The ViewModeProvider Pattern

```typescript
type ViewMode = 'card' | 'list'

interface ViewModeContextType {
  viewMode: ViewMode           // Current view mode
  toggleViewMode: () => void   // Function to toggle between modes
}
```

**Why this pattern?**
- Encapsulates state logic in one place
- Provides a clean API (`useViewMode()`) for consumers
- Throws error if used outside provider (catches mistakes early)

## Implementation Hints

<Accordion title="How do I create a Context?">

Use `createContext` with a type and initial value:

```typescript
const ViewModeContext = createContext<ViewModeContextType | null>(null)
```

Initial value is `null` because context should only be accessed within the provider.

</Accordion>

<Accordion title="Why create a custom useViewMode hook?">

It provides:
1. Better error messages ("must be used within provider")
2. Type safety (no need to check for null everywhere)
3. Cleaner import syntax

```typescript
// With custom hook
const { viewMode, toggleViewMode } = useViewMode()

// Without custom hook (verbose)
const context = useContext(ViewModeContext)
if (!context) throw new Error('...')
const { viewMode, toggleViewMode } = context
```

</Accordion>

<Accordion title="How do I toggle between two values?">

Use a ternary operator in the state setter:

```typescript
const toggleViewMode = () => {
  setViewMode(prev => prev === 'card' ? 'list' : 'card')
}
```

This pattern works for any binary toggle.

</Accordion>

<Accordion title="When should I use Context vs Props?">

**Use Context when:**
- Value is needed by many components at different nesting levels
- Passing props through multiple levels becomes cumbersome
- Value represents global/shared state (theme, auth, view mode)

**Use Props when:**
- Only parent and direct children need the value
- Value is specific to a component instance
- You want component to be reusable in different contexts

</Accordion>

## Common Issues

<Tip>
**Issue:** "Cannot read property 'viewMode' of null"

**Solution:** Make sure you wrapped your app with `ViewModeProvider`:

```typescript
<ViewModeProvider>
  <App />
</ViewModeProvider>
```
</Tip>

<Tip>
**Issue:** Descriptions always show or never show

**Solution:** Check the conditional rendering logic:

```typescript
{viewMode === 'list' && product.description && (
  <p>{product.description}</p>
)}
```

Both conditions must be true for description to render.
</Tip>

<Tip>
**Issue:** Toggle state doesn't persist when navigating

**Solution:** This is expected! Context state resets when the provider unmounts. If you need persistence, combine Context with `localStorage`:

```typescript
const [viewMode, setViewMode] = useState<ViewMode>(() => {
  return (localStorage.getItem('viewMode') as ViewMode) || 'card'
})

const toggleViewMode = () => {
  setViewMode(prev => {
    const next = prev === 'card' ? 'list' : 'card'
    localStorage.setItem('viewMode', next)
    return next
  })
}
```
</Tip>
